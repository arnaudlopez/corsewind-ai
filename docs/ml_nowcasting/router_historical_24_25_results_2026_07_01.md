## Decision

Using all 2024/2025 data is the right direction, but the current historical
artifact is incomplete for the production router because it lacks the champion
rail.

Do not promote the historical router.

Keep:

```text
champion -> median p50
strong_gated -> shadow strong-wind rail
raw/foundation/probability heads -> alert/risk context
router -> research until trained on out-of-fold champion + strong_gated rails
```

## Next Required Dataset

```text

```

The key requirement is that the champion/strong_gated prediction for each
historical row must be generated by a model that did not train on that row.
That means rolling-origin or monthly/quarterly out-of-fold prediction
generation, not a single model trained on all 2024/2025 and then scored on
2024/2025.
